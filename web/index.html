Критерии оценки:

  1. Правильная структура документа, страница проходит валидацию (максимум 3 балла)
  2. Есть заголовок, параграф, изображение, список, ссылка (5 баллов)
  3. Подключен отдельный стилевой файл, есть фон страницы. Подключена иконка (3 балла)
  4. Есть семантическая структура: использованы семантические теги или div с классами (балл за каждый элемент структуры, включая шапку и подвал с годом и копирайтом, максимум 4 балла)
  5. Использованы вложенные блоки (2 балла)
  6. Сайт состоит из нескольких страниц, каждая из которых имеет меню. Меню сделано на основе списка (5 баллов максимум)
  7. Использован хотя бы один из JS-скриптов, включая тест, показ и исчезновение блока по действию (5 баллов максимум)
  8. Подключены внешние шрифты (3 балла максимум)
  9. Установлен фон для нескольких блоков на странице, установлен размер блоков во весь экран или по размеру картинки (максимум 2 балла)
  10. Есть отдельная "лендинг-страница", состоящая из нескольких слайдов с заголовками, блоками и текстом (максимум 5 баллов)
  11. Сайт выложен на хостинг (максимум 3 балла, github приветствуется)
  12. Баллы за количество страниц (обсуждается: при наличии нескольких сайтов ставится лучший из поставленных баллов по отдельной категории)
  13. Сайт сверстан с учетом мобильных устройств (адаптивная верстка). Максимум 5 баллов

Инструменты

Для того, чтобы создавать и просматривать web-страницы, нужно всего две программы: текстовый редактор и web-браузер. Хорошим выбором будет Microsoft Visual Studio Code (есть поддержка русского языка в интерфейсе) и Google Chrome. Если VSCode ощутимо "тормозит", подойдет более простой редактор Notepad++. В качестве браузера также подойдут Apple Safari, Mozilla Firefox и Microsoft Edge. Использование Microsoft Internet Explorer нежелательно, так как он устарел и не поддерживает должным образом многие современные технологии.

В текстовом редакторе создаются файлы с расширением html и css. Их необходимо сохранить на диске. Затем файл html открывается в браузере при помощи диалогового окна, вызываемого нажатием клавиш Ctrl+O. Для просмотра исходного кода страницы следует нажать Ctrl+U. Визуальный инспектор кода вызывается сочетанием клавиш Ctrl+Shift+C. Консоль разработчика доступна при нажатии F12.

В дальнейшем файлы web-проекта могут быть выложены на хостинг для доступа к ним по сети Интернет через URL, уникальный адрес. Хостинг может быть бесплатным, как, например, 000webhost.com или GearHost: .NET and PHP Cloud Hosting, еще (список). После регистрации становится возможным выкладывать свои web-страницы в открытый доступ.

Для быстрой проверки кода подойдут онлайн-сервисы типа Codepen.io. После регистрации своими примерами можно делиться!
В окно HTML вставляется код из тега <body>. В окно CSS копируется содержимое стилевого файла и/или содержимое html-тега <style>. Окно JavaScript можно скрыть.

Основы HTML

Web-страницы созаются на языке гипертекствой разметки, HyperText Markup Language. В HTML для разметки пользуются тегами – ключевыми словами в угловых скобках. Теги бывают одиночные и парные. Парные теги являются контейнерами, в которых можно расположить текст, изображения или иной объект. Web-дизайнер должен понимать смысл и назначение основных тегов html.
